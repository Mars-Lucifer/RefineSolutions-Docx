## Категории записей базы знаний

Для упрощения структуры и ускорения добавления материалов используется минимальный набор универсальных категорий.  
Тип записи определяется папкой, в которой она находится.

### (1) Проблемно-ориентированные записи

Записи о технических проблемах, сбоях и инцидентах, а также способах их решения.

Используется для:
- багов и некорректной логики
- ошибок среды и инфраструктуры
- production-инцидентов

> Пример: после обновления зависимостей перестала работать авторизация — описание симптомов, причины и финального решения.

---

### (2) Процессные стандарты

Описание рабочих процессов, стандартов и правил работы внутри студии.

Используется для:
- регламентов разработки
- процессов работы с клиентами
- внутренних правил команды

> Пример: как проходит Code Review — этапы проверки, требования к pull request и правила согласования.

---

### (3) Справочные материалы

Справочные материалы для быстрого доступа к информации без обучения.

Используется для:
- FAQ
- API и endpoints
- команды и технические шпаргалки

> Пример: список основных npm-команд проекта с кратким объяснением и примерами использования.

---

### (4) Код-ориентированные записи

Переиспользуемый код и технические заготовки.

Используется для:
- готовых сниппетов
- шаблонов конфигураций
- повторяемых технических решений

> Пример: базовый API обработчик с авторизацией и проверкой ошибок для всех frontend-проектов студии.

---

### (5) Архитектурные решения

Зафиксированные технические и архитектурные решения с обоснованием выбора.

Используется для:
- архитектурных решений
- выбора технологий
- анализа альтернатив

> Пример: выбор Next.JS как основного фреймворка, почему он, первичная настройка и структура.
